Neural Networks and Deep Learning: Architectures and Mechanisms

The Perceptron: The Building Block

The perceptron, invented by Frank Rosenblatt in 1958, is the simplest neural network unit. It takes multiple numerical inputs, multiplies each by a weight, sums them, adds a bias, and passes the result through an activation function to produce an output. Formally: output = activation(sum(weight_i * input_i) + bias).

A single perceptron can only learn linearly separable patterns. The famous XOR problem — which requires a non-linear decision boundary — cannot be solved by a single perceptron, motivating the development of multi-layer networks.

Multi-Layer Networks (MLPs)

A Multi-Layer Perceptron (MLP) consists of:
- An input layer that receives features
- One or more hidden layers that learn intermediate representations
- An output layer that produces predictions

Each layer is fully connected to the next (hence also called "fully connected" or "dense" layers). The ability to stack multiple non-linear transformations allows MLPs to learn complex patterns that a single perceptron cannot.

Activation Functions

Activation functions introduce non-linearity into neural networks, enabling them to learn complex relationships. Without non-linearity, a deep network would collapse to a single linear transformation.

Sigmoid: Outputs a value between 0 and 1. Historically popular but suffers from the vanishing gradient problem in deep networks — gradients become extremely small, slowing learning.

Tanh (Hyperbolic Tangent): Outputs between -1 and 1. Zero-centered, which can make optimization easier, but still suffers from vanishing gradients.

ReLU (Rectified Linear Unit): Defined as max(0, x). It is the most widely used activation function today because it is computationally efficient and does not saturate for positive values. However, it can suffer from the "dying ReLU" problem where neurons output zero for all inputs.

Leaky ReLU: A variant of ReLU that allows a small gradient (e.g., 0.01x) for negative inputs, addressing the dying ReLU problem.

GELU (Gaussian Error Linear Unit): Used in modern Transformer models like BERT and GPT. It approximates the expected value of ReLU with stochastic regularization, providing smoother gradients.

Softmax: Applied to the output layer for multi-class classification. It converts raw scores (logits) into a probability distribution where all probabilities sum to 1.

Backpropagation: How Neural Networks Learn

Backpropagation is the algorithm used to train neural networks. It works by:

1. Forward Pass: Input data is passed through the network to produce a prediction.
2. Loss Computation: The prediction is compared to the true label using a loss function (e.g., cross-entropy for classification, MSE for regression).
3. Backward Pass: The gradient of the loss with respect to each weight is computed using the chain rule of calculus.
4. Weight Update: Weights are updated in the direction that decreases the loss, typically using gradient descent or an adaptive optimizer like Adam.

The chain rule allows gradients to flow backward through the layers. The key insight is that if you know the gradient at a layer, you can compute the gradient at the previous layer by multiplying by the local derivative.

Optimizers

Stochastic Gradient Descent (SGD): Updates weights using the gradient computed on a small mini-batch of data. Momentum SGD adds a velocity term to accelerate convergence.

Adam (Adaptive Moment Estimation): The most popular optimizer for deep learning. It adapts the learning rate for each parameter using estimates of first and second moments of the gradients.

AdamW: Adam with weight decay decoupled from the gradient update, which improves generalization. Used extensively in training large language models.

Convolutional Neural Networks (CNNs)

CNNs are specialized architectures designed for processing grid-like data such as images. Their key components are:

Convolutional Layers: Apply learnable filters (kernels) that slide across the input to detect local features like edges, textures, and shapes. Each filter produces a feature map.

Pooling Layers: Reduce spatial dimensions using max pooling or average pooling, providing translation invariance and reducing computation.

Fully Connected Layers: At the end of the network, flattened feature maps are passed through dense layers for classification.

CNNs are the backbone of computer vision tasks including image classification (ResNet, VGG), object detection (YOLO, Faster R-CNN), and image segmentation (U-Net, DeepLab).

Recurrent Neural Networks (RNNs)

RNNs are designed for sequential data where order matters, such as text, audio, and time series. They maintain a hidden state that captures information from previous time steps.

The vanishing gradient problem is severe in vanilla RNNs — gradients decay exponentially over long sequences, making it hard to learn long-range dependencies.

Long Short-Term Memory (LSTM): Solves the vanishing gradient problem with a gating mechanism. It uses three gates — forget, input, and output — to control what information is retained or discarded across time steps.

Gated Recurrent Unit (GRU): A simplified version of LSTM with two gates (reset and update). It is faster to train while achieving comparable performance on many tasks.

The Transformer Architecture

Introduced in the 2017 paper "Attention Is All You Need" by Vaswani et al., the Transformer architecture has largely replaced RNNs for NLP tasks and is the foundation of modern large language models.

Key components of the Transformer:

Self-Attention Mechanism: Allows each token in a sequence to attend to all other tokens, capturing both local and global relationships simultaneously. Attention is computed as:
  Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) * V
where Q (queries), K (keys), and V (values) are linear projections of the input.

Multi-Head Attention: Runs the self-attention mechanism multiple times in parallel with different learned projections, allowing the model to attend to information from different representation subspaces.

Positional Encoding: Since Transformers have no inherent notion of sequence order (unlike RNNs), positional encodings are added to input embeddings to inject positional information.

Feed-Forward Layers: After attention, each position passes through the same two-layer feed-forward network with ReLU or GELU activation.

Layer Normalization and Residual Connections: Applied around each sub-layer to stabilize training and enable very deep networks.

The Encoder processes the input sequence into contextual representations. The Decoder generates output tokens autoregressively, attending to both previous outputs and the encoder output.

BERT (Bidirectional Encoder Representations from Transformers) uses only the encoder and is pre-trained using masked language modeling. GPT models use only the decoder and are trained autoregressively to predict the next token. Both architectures can be fine-tuned on downstream tasks.

Training Deep Networks: Practical Considerations

Batch Normalization: Normalizes activations within a mini-batch during training, which stabilizes training, allows higher learning rates, and acts as a regularizer.

Dropout: During training, randomly sets a fraction of neurons to zero, preventing co-adaptation of neurons and reducing overfitting.

Weight Initialization: Proper initialization (e.g., Xavier/Glorot for sigmoid/tanh, He initialization for ReLU) prevents vanishing or exploding gradients at the start of training.

Learning Rate Scheduling: Strategies like cosine annealing, warm-up schedules, and learning rate decay help the optimizer converge to better minima.

Transfer Learning: Pre-trained models (e.g., ImageNet-trained ResNet, BERT, GPT) can be fine-tuned on specific tasks with far less data and computation than training from scratch. This is one of the most impactful techniques in modern deep learning.
