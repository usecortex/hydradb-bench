# Skill: Analyze Benchmark Reports

## Description
Read, interpret, and compare benchmark reports generated by the framework.

## When to Use
When the user asks to review results, interpret scores, compare providers, identify weak areas, or extract insights from benchmark reports.

## Report Locations
All reports are in `./reports/` directory (configurable via `reporting.output_dir`).

### File Types
- `<run_id>_<provider>.json` — Full machine-readable results
- `<run_id>_<provider>.html` — Interactive HTML report
- `<run_id>_<provider>.csv` — Flattened per-sample data
- `<run_id>_comparison.html` — Side-by-side HydraDB vs Supermemory comparison

## How to Analyze

### Quick Overview
```bash
# List all reports
ls -la reports/

# Get aggregate scores from a JSON report
python -c "
import json
r = json.load(open('reports/<run_id>_hydradb.json'))
print(f'Samples: {r[\"total_samples\"]}, Errors: {r[\"error_count\"]}')
for m, s in r['aggregate_scores'].items():
    print(f'  {m}: {s:.4f}')
print(f'Latency p50: {r[\"latency_p50\"]:.0f}ms, p95: {r[\"latency_p95\"]:.0f}ms')
"
```

### Convert JSON to CSV for Spreadsheet Analysis
```bash
python json_to_csv.py reports/<report>.json
# or specify output path:
python json_to_csv.py reports/<report>.json --output reports/analysis.csv
```

### Identify Weak Samples
Look at `per_sample` entries where scores are low (< threshold) or `None` (errors/timeouts):
```python
import json
r = json.load(open('reports/<run_id>_hydradb.json'))
for s in r['per_sample']:
    for metric, score in s['scores'].items():
        if score is not None and score < 0.5:
            print(f"LOW: {s['sample_id']} | {metric}={score} | Q: {s['question'][:80]}")
```

## Score Interpretation
| Score Range | Meaning |
|-------------|---------|
| >= 0.8 | Strong performance (green in HTML reports) |
| 0.5 - 0.8 | Moderate, room for improvement (yellow) |
| < 0.5 | Weak, needs attention (red) |
| None/N/A | Error or timeout during evaluation |

## Key Metrics Explained
| Metric | What It Measures |
|--------|-----------------|
| `answer_accuracy` | Does the generated answer match the reference? (binary 0/1) |
| `contextual_precision` | Are relevant contexts ranked higher than irrelevant ones? |
| `contextual_recall` | Were all relevant contexts successfully retrieved? |
| `contextual_relevancy` | Is the retrieved context relevant to the question? |
| `answer_relevancy` | Is the answer relevant to the question asked? |
| `faithfulness` | Is the answer grounded in the retrieved context? |
| `hallucination` | Does the answer contain information not in context? |

## Comparing Providers
When `--provider both` is used, a comparison HTML is generated. To programmatically compare:
```python
import json
h = json.load(open('reports/<run_id>_hydradb.json'))
s = json.load(open('reports/<run_id>_supermemory.json'))
for metric in h['aggregate_scores']:
    hs, ss = h['aggregate_scores'][metric], s['aggregate_scores'].get(metric)
    print(f"{metric}: HydraDB={hs:.4f} Supermemory={ss:.4f if ss else 'N/A'}")
```
